# Project Context: CodeReview Automation Platform

## 🏗️ Architecture Overview
This is a microservices-based platform designed to automate code analysis for GitHub repositories.

### 🚀 Services
- **Frontend:** React 19 (Nginx) - Port 3000 (Host) / 80 (Container).
- **Orchestrator:** .NET 9 API - Port 5116 (Host) / 5116 (Container). Handles GitHub integration and persistence.
- **Analyzer:** PHP 8.2 Slim - Port 8000 (Internal). Performs static analysis (Complexity, Style, Security).
- **Database:** MySQL 8.0 - Port 3306 (Internal).
- **Monitoring:** Prometheus & Grafana stack for observability.

## 🛡️ Security & Infrastructure State
The project has undergone significant hardening:

1.  **Secret Management (HashiCorp Vault):** 
    - Plaintext secrets have been removed from K8s manifests.
    - Vault is deployed in `vault` namespace (Dev Mode).
    - **Vault Secrets Operator (VSO)** manages synchronization.
    - Secrets are seeded via `scripts/seed-vault.sh`.
    - Applications consume secrets via native K8s Secrets generated by `VaultStaticSecret` CRDs.

2.  **Supply Chain (Trivy):**
    - CI pipeline in `.github/workflows/ci.yml` scans images for `CRITICAL` vulnerabilities.
    - Pushes to Docker Hub and `deploy` branch updates are gated by security scan results.

3.  **Infrastructure as Code (Terraform):**
    - Terraform manages Namespaces, ServiceAccounts, RBAC, Vault Server, and VSO configuration.
    - **Ownership Split:** Terraform owns the Platform; ArgoCD owns the Application Runtime.

## 🛠️ Linux Machine Setup Instructions
Once the code is cloned to the Linux machine:

1.  **Kind Cluster:** Create using `kind create cluster --config kind-config.yaml`.
2.  **Vault CLI:** Ensure the `vault` binary is in your PATH.
3.  **Terraform Apply:**
    - Navigate to `terraform/environments/kind/`.
    - Run `terraform init`.
    - Run the targeted apply: `terraform apply -target=module.namespaces -target=module.service_accounts -target=module.prometheus_rbac -target=module.vault -target=module.vault_secrets_operator.helm_release.vso`.
4.  **Seeding:** Execute `bash scripts/seed-vault.sh`.
5.  **Final Apply:** 
    - Set environment variables: `export TF_VAR_vault_address="http://127.0.0.1:30200"` and `export TF_VAR_vault_token="root"`.
    - Run `terraform apply`.
6.  **Deploy Apps:** Run `kubectl apply -f k8s/base/`.
